## ⚠️ 시스템 요구사항

- **운영체제**: macOS 10.15 (Catalina) 이상
- **프로세서**: Apple Silicon (M1, M2, M3 등 aarch64 아키텍처)
- **메모리**: 최소 4GB RAM 권장
- **저장공간**: 최소 500MB 여유 공간

> **참고**: Intel Mac 에서는 실행되지 않습니다. Intel Mac 사용자는 `x86_64` 버전이 필요합니다.

## 📥 설치 방법

### 1 단계: 파일 다운로드
제공받은 `Kuse Cowork_0.1.0_aarch64.dmg` 파일을 다운로드합니다.

### 2 단계: DMG 파일 열기
다운로드한 DMG 파일을 더블클릭하여 마운트합니다.

```
Kuse Cowork_0.1.0_aarch64.dmg → 더블클릭
```

### 3 단계: Applications 폴더로 이동
DMG 창이 열리면 다음 중 하나를 수행합니다:

**방법 A**: `Kuse Cowork` 아이콘을 드래그하여 `Applications` 폴더 아이콘 위에 놓습니다.

**방법 B**: 터미널에서 다음 명령어 실행:
```bash
cp -R "/Volumes/Kuse Cowork/Kuse Cowork.app" /Applications/
```

### 4 단계: 애플리케이션 실행
1. Finder 에서 `Applications` 폴더로 이동합니다.
2. `Kuse Cowork` 앱을 더블클릭하여 실택합니다.

### 5 단계: 보안 경고 처리 (첫 실행 시)
macOS 의 보안 기능으로 인해 첫 실행 시 경고가 표시될 수 있습니다:

```
"Kuse Cowork"는 개발자를 확인할 수 없으므로 열 수 없습니다.
```

이 경우 다음 단계로 진행하세요:

1. **시스템 설정** → **개인정보 보호 및 보안**으로 이동
2. 아래로 스크롤하여 "Kuse Cowork" 관련 메시지 확인
3. **그래도 열기** 버튼 클릭
4. 확인 팝업에서 **열기** 클릭

앱이 정상적으로 설치되었는지 확인하는 방법:


## ⚙️ 첫 실행 후 설정

현재 배포 preset 기준으로 앱을 처음 열면 아래 값이 미리 들어가 있습니다.

### LLM 기본값

- Provider: `custom`
- Model: `Qwen3.5-397B-A17B-FP8`
- Base URL: `https://llm.ss-fai.cloud/v1`
- Max Tokens: `4096`
- Temperature: `0.7`

포함되지 않는 값:

- API key
- provider 별 저장 키

해야 할 일:

1. **Settings** 열기
2. **Custom** 탭이 선택되어 있는지 확인
3. API key 입력
4. 필요하면 model 또는 base URL 수정
5. **Test Connection** 실행

### MCP 기본값

현재 MCP preset 으로 아래 서버들이 미리 추가되어 있습니다.

- Name: `Gitlab MCP`
- URL: `https://mcp.ss-fai.cloud/gitlab/mcp`
- Auth Type: `Bearer Token`
- Name: `Mattermost MCP`
- URL: `https://mcp.ss-fai.cloud/mattermost/mcp`
- Auth Type: `Bearer Token`
- Name: `Outline MCP`
- URL: `https://wiki.ss-fai.cloud/mcp`
- Auth Type: `Bearer Token`

포함되지 않는 값:

- Bearer token
- OAuth client secret
- custom header 값

해야 할 일:

1. **MCP Settings** 열기
2. `Gitlab MCP`, `Mattermost MCP`, `Outline MCP` 중 사용할 서버 선택
3. 각 서버에 맞는 Bearer token 입력
4. 저장 후 **Connect** 실행

## ❓ 문제 해결

### Q1. "앱이 손상되었으므로 열 수 없습니다" 메시지가 표시됩니다.

**해결 방법**:
```bash
# 터미널에서 다음 명령어 실행
xattr -cr /Applications/Kuse\ Cowork.app
```

### Q2. Intel Mac 을 사용하고 있습니다.

이 DMG 파일은 Apple Silicon 전용입니다. Intel Mac 용 `x86_64` 버전이 필요합니다.

프로세서 확인 방법:
```bash
uname -m
```
- `arm64`: Apple Silicon (이 파일 사용 가능)
- `x86_64`: Intel Mac (다른 버전 필요)

### Q3. 앱이 응답하지 않습니다.

1. 앱을 강제 종료 (⌘ + ⌥ + Esc)
2. 다음 폴더의 캐시 파일 삭제:
   ```bash
   rm -rf ~/Library/Caches/com.kuse.cowork
   rm -rf ~/Library/Application\ Support/kuse-cowork
   ```
3. 앱 재실행

### Q4. DMG 파일이 열리지 않습니다.

파일이 손상되었을 수 있습니다. 파일을 다시 다운로드받으세요.
